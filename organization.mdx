You organize documentation effectively by using folders, tags, and hierarchies in Paperguide. This keeps content accessible and maintainable as projects grow. Start with a clear outline to avoid clutter.

## Tagging and Categorization

Apply tags like `api`, `tutorial` to pages. You search by multiple tags for precise results.

<Tabs>
  <Tab title="Adding Tags" icon="plus">
    In the editor, select pages and assign from the tag panel.
    ```javascript
    // API example for bulk tagging
    const pages = ['doc1', 'doc2'];
    const tags = ['api', 'v1'];
    await fetch('/api/pages/tag', {
      method: 'POST',
      body: JSON.stringify({ pages, tags })
    });
    ```
  </Tab>
  <Tab title="Searching" icon="search">
    Use the search bar: `tag:api status:draft`.
    ```bash
    # CLI integration example
    paperguide search --tags api --limit 20
    ```
  </Tab>
</Tabs>

<Steps>
  <Step title="Create Outline" icon="list">
    Map your doc structure on paper or in a mind map tool.
  </Step>
  <Step title="Implement Folders" icon="folder-plus">
    Drag pages into folders via the sidebar.
  </Step>
  <Step title="Review Structure" icon="check">
    Ensure logical flow from general to specific topics.

</Steps>

## Maintenance Best Practices

Regularly audit your organization to keep it efficient. You archive outdated pages and update links.

<ExpandableGroup>
  <Expandable title="Handling Large Projects" default-open="true">
    Split into multiple workspaces if exceeding `>500` pages.
  </Expandable>
  <Expandable title="Migration Tips" default-open="false">
    Export from old tools and import via CSV for metadata.
  </Expandable>
</ExpandableGroup>
